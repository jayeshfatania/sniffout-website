# Part 1: Competitive Research

## 1. AllTrails (alltrails.com)

**Design partner:** Red Antler (rebrand)

**Primary colours:**
- `#94f477` - signature neon lime green. Used for CTA buttons and map route lines. This is the single most recognisable colour in the outdoor app space. It reads as energetic, startup-forward, almost electric.
- `#161f13` - near-black forest tone, used for primary text
- `#1b2818` - deep forest green for dark surfaces
- `#f2eee7` - warm off-white background (the "birch" token - nearly identical to Sniffout's `#F7F5F0`)
- `#c1af98` - warm sand secondary accent

**Typography:**
- Custom proprietary font: **AllTrailsAeonik** throughout. No system or Google fonts. Custom ownership creates instant recall.

**Hero treatment:**
Full-bleed cycling photography with a search bar overlaid. Trail and landscape imagery, aspirational but not intimidating. The photography does the work.

**Card design:**
Clean white cards. Difficulty badges colour-coded in green/yellow/red. Distance and elevation stats shown prominently. Map interface uses the neon-green route lines on cartographic tiles.

**Instant recognisability:**
The neon lime on CTAs and map routes. You cannot mistake it. The custom typeface reinforces ownership. The product is technically accomplished and visually confident.

**What Sniffout must not copy:**
The neon green. Full-bleed photo carousels as the primary homepage hero. Dark forest green for page backgrounds and dark-mode components (too close in palette register to Sniffout's woodland green).

## 3. Walk Highlands (walkhighlands.co.uk)

**Primary colours:**
- `#2d4461` - mid-toned slate blue. Primary nav background and dominant UI colour.
- `#204b79`, `#4b6d9a` - a family of steel blues
- `#ffc200` - amber yellow for focus states and accent highlights

**Typography:**
No custom fonts. System fonts only: Verdana, Arial, Trebuchet MS, Georgia as fallbacks. Deliberately minimal.

**Hero treatment:**
Functional text-led hero. No full-bleed photography on the homepage. Region-based navigation grids above the fold. Utility over aesthetics throughout.

**Card design:**
List-based walk entries with text metadata. No card shadows, no modern styling. Walk stats shown as plain text rows.

**Instant recognisability:**
The absence of design ambition is the signal. It reads as a trusted community resource built by walkers for walkers, not a designed product. It is the most visited walking site in Scotland precisely because it prioritises content depth over visual form.

**What Sniffout must not copy:**
The functional-to-a-fault approach is not aspirational. But the depth and trust signals built through content are worth noting. Walk Highlands proves that even basic design can win on content quality alone.

# Part 2: Identity Recommendations

---

## 1. Typography

**Current state:** Plus Jakarta Sans is the sole typeface at all weights and sizes. It is an excellent, versatile, modern sans-serif. The problem is that it provides no differentiation at display sizes (H1, hero headlines, featured article titles). It reads as competent but generic when used at 28px+ because many sites use it.

**Recommendation: Introduce Fraunces as a display heading typeface for H1 and featured article titles only.**

Fraunces is a variable font on Google Fonts by Undercase Type. It was commissioned specifically to have warmth, optical character, and a literary sensibility at large sizes. It has a "wobbly" humanist quality that reads as considered and personal rather than tech-startup. It is widely available but has not been adopted in the outdoor app or dog walking category.

**Pairing logic:**
- **Fraunces 700** (display/H1): Walk page titles, homepage headline, article H1s, guide hero titles
- **Plus Jakarta Sans 400-800** (everything else): Nav, body copy, UI components, CTAs, metadata, section labels

**Why this differentiates from AllTrails:**
AllTrails uses a clean modern grotesque (AeonikAllTrails). Komoot uses a geometric display font (Nohemi). Every outdoor app in the category uses sans-serifs exclusively. Introducing a warm display serif creates immediate visual distinction. When someone lands on a Sniffout walk page and reads "Richmond Park Dog Walk" in Fraunces 700, it will feel instantly unlike AllTrails, Komoot, or any other outdoor discovery product.

**The serif/sans split also carries meaning:** serif = curated, considered, editorial (the walk is a destination worth writing about); sans = clear, functional, modern (the UI chrome stays approachable).

**Exact values:**
```
Google Fonts: Fraunces
Weights to load: 700 (display use only)
CSS: font-family: 'Fraunces', Georgia, serif
Apply to: h1.walk-title, h1.guides-h1, .home-headline, .guide-hero-title, .guide-article-h1
Keep Plus Jakarta Sans: all other elements unchanged
```

**Import addition to head.html:**
```html
<link href="https://fonts.googleapis.com/css2?family=Fraunces:opsz,wght@9..144,700&family=Plus+Jakarta+Sans:wght@400;500;600;700;800&display=swap" rel="stylesheet">
```

The `opsz` axis (optical size 9 to 144) means the letterforms will automatically adjust for legibility: more expressive at 144pt display sizes, more conservative and legible at smaller sizes. At 22px mobile / 28px desktop for walk titles, it will read warm and distinctive without being decorative.

## 2. Colour

**Current state:** `#2C4A14` is a deep woodland green used as the primary brand colour. It is darker and more mature than AllTrails' neon lime `#94f477`. The two greens are not confusingly similar - the issue is more subtle. The current palette has no warm accent colour that is distinctly Sniffout's, and the amber in the system (`#D97706`) is restricted to warning usage.

**The green itself does not need to change.** `#2C4A14` is distinctive within the category. Walk Highlands uses slate blue. AllTrails owns neon lime. Kennel Club uses teal. Komoot uses olive. `#2C4A14` sits in its own territory: darker, richer, more like ink from a pressed leaf than a sports brand. Keep it.

**Recommendation 1: Introduce earthy sienna `#B85C2C` as a secondary brand accent.**

Current status: no secondary brand accent exists. Amber is in the system but locked to warning states. This means every non-warning accent defaults to the same woodland green, creating a monochromatic risk.

`#B85C2C` is a warm earthy sienna. It references autumn woodland, muddy fields after rain, terracotta pots on a garden path. It is distinctive within the outdoor app category (no competitor uses it), it pairs beautifully with `#2C4A14`, and it carries warmth and personality.

Use cases for the sienna accent:
- "Get the app" header CTA pill (replaces the current green-on-green pill with more contrast and personality)
- Featured walk badge / "Sniffout Pick" badge
- Pull-quote or callout highlights in guide articles
- Hover states on navigation links

```css
--accent: #B85C2C;
--accent-light: #FAF0E8;
--accent-border: #E8A87C;
```

**Recommendation 2: Increase boldness of brand green usage.**

Currently, the woodland green `#2C4A14` appears most prominently in the header, CTAs, and chips. On page bodies, it retreats. The Monzo principle applies here: own your colour by using it consistently and bravely. Specifically:

- The homepage hero section is correctly full-bleed brand green. This should stay and be protected.
- The walk hero gradient currently fades from `#C8D8B0` to `#6E9048` - these lighter greens feel generic. Consider darkening the gradient to anchor it more clearly in `#2C4A14` territory, or replace the gradient with a flat colour section for the hero band.
- Section labels (`.walk-section-heading`, 10px uppercase) currently use `#888888`. Consider using `#2C4A14` at low opacity (`color: rgba(44,74,20,0.55)`) so they remain quiet but carry the brand colour into body text areas.

**Exact new token:**
```css
--accent: #B85C2C;
--accent-light: #FAF0E8;
--accent-border: #E0A078;
```

## 3. Card design

**Current state:** The walk card (`walk-card-h`, `walk-list-card`) uses a full-bleed gradient image area at the top, with text content below. When photos are available, the image will fill the top of the card. This is structurally different from AllTrails' overlay approach (text on top of photo) but uses the same compositional logic: hero image is the dominant visual, content sits beneath it.

**The current structure is already more editorial than AllTrails.** AllTrails overlays title and metadata directly on the photo with a dark gradient. Sniffout has the image as a header band and the text below in a separate white body. This is the correct direction. The recommendations are refinements, not rebuilds.

**Recommendation 1: Add a subtle depth signal to walk cards.**

Currently cards use `border: 0.5px solid var(--border)` with no shadow. This is intentionally flat. To add a sense of quality without adding heavy box-shadows (which feel dated), introduce a very subtle `box-shadow` on walk list cards only (not on the hero card, not on guide cards):

```css
.walk-list-card {
  box-shadow: 0 1px 4px rgba(0,0,0,0.05), 0 0 0 0.5px rgba(0,0,0,0.06);
}
```

This is a 5% opacity 1px blur - barely perceptible on individual cards but creates a sense of the cards "sitting on" the background when the grid is seen as a whole. This is a hallmark of premium mobile-first products (Airbnb, Monzo app cards) and makes the grid feel curated rather than listed.

**Recommendation 2: Add the accent colour to walk difficulty badges.**

The difficulty badge (currently rendered via `.walk-hero-badge--left`) shows the difficulty as white-background text. Introduce colour variants that use the accent sienna for "Moderate" and a deeper tone for "Hard":

```css
.walk-hero-badge--moderate { background: #FAF0E8; color: #B85C2C; }
.walk-hero-badge--hard { background: #FEF2F2; color: #B91C1C; }
```

The easy badge retains white/green. This visual differentiation signals care and specificity. It is a small detail that experienced users notice and trust.

**Recommendation 3: When real photos arrive, do not use a dark overlay.**

AllTrails overlays dark gradients on photo cards with text on top. When Sniffout adds real walk photography, keep the image-above / text-below structure that already exists. The photo should be a preview, not a canvas for text. This is the primary structural differentiator from AllTrails' card pattern.

## 4. Hero treatment

**Current state:** The homepage uses a full-bleed brand green hero with centred headline, supporting text, and a CTA button. Luna's personal dog photos are referenced as the hero content in the brief but the current live site does not use photography in the hero.

**Assessment of the current hero:** The brand green full-bleed approach is actually strong and distinctive. It does not look like AllTrails. It looks closer to Headspace (who also uses a solid brand colour hero rather than photography). Keep this structure.

**Recommendation 1: Use Luna's photos in a specific, editorial way rather than as a carousel.**

Do not introduce a photo carousel. Carousels are:
- Associated with AllTrails and generic outdoor apps
- Poor on mobile (scroll conflict, low completion rate)
- Dilute the focus of a conversion-first hero

Instead, introduce a single full-bleed photo moment in a dedicated section below the hero. Call this the "Meet the walks" section or similar. A single large photograph of a dog on a specific walk, with the walk name overlaid at the bottom in a clean pill. This is closer to how premium editorial photography is used: one considered image rather than a slideshow.

**Recommendation 2: Add a subtle warmth texture to the brand green hero.**

The flat `#2C4A14` hero is clean but slightly cold at the edges. A very subtle noise texture (2-3% opacity grain overlay using a CSS pseudo-element with `background-image: url('data:image/svg+xml...')` or a tiny PNG) adds tactility and warmth to the brand green surface. This is a technique used by Stripe, Loom, and Headspace to give flat colour backgrounds a premium "hand-crafted" quality.

A noise texture overlay at `opacity: 0.03` to `0.04` is imperceptible in a screenshot but makes the surface feel warmer when experienced on screen.

```css
.home-hero::before {
  content: '';
  position: absolute;
  inset: 0;
  background-image: url('/img/noise.png');
  background-repeat: repeat;
  background-size: 200px;
  opacity: 0.035;
  pointer-events: none;
}
```

A 200x200px monochrome noise PNG at roughly 50% grain density serves as the texture tile.

## 5. Personality markers

These are the 5 specific design decisions that, taken together, would make Sniffout instantly recognisable and immediately different from AllTrails.

---

**PM-1: Fraunces display serif for all H1 headings.**

When a walk title reads "Richmond Park Dog Walk" in Fraunces 700, it signals: this is a considered, curated product, not an algorithm-generated listing. The warm serif letterforms suggest the walk has been written about, not scraped. No outdoor discovery app uses a serif. This is the single highest-impact differentiator available for zero cost (Google Fonts, free).

---

**PM-2: Earthy sienna `#B85C2C` as the secondary accent.**

A second brand colour that is distinctly "Sniffout" and carries no AllTrails association. Sienna reads as: autumn mud, terracotta pots, warm clay. It grounds the brand in the physical world of dog walking rather than the digital world of sports tracking. Applied to the "Get the app" header CTA and the "Sniffout Pick" walk badge, it provides a second visual note that signals personality.

---

**PM-3: The "For your dog" green box as a signature component.**

The `.walk-dog-box` component - the green-tinted box with "FOR YOUR DOG" label that appears on every walk page - is a genuinely original component. No competitor has a dedicated dog-specific information block in their walk detail pages. This component should be visible in marketing materials, previewed in homepage cards, and treated as a signature Sniffout element. It communicates the core brand premise at a glance: this is for dog owners specifically, not generic hikers.

---

**PM-4: Section labels in brand green at low opacity, not grey.**

Currently `.walk-section-heading` uses `color: #888888`. This is common and correct for utility but misses an opportunity. Change the section label colour to `rgba(44,74,20,0.55)` - a mid-tone version of the brand green. This means every time you scan a Sniffout page, the structural labels carry the brand colour at a quiet register. It is a subtle brand application that accumulates across the page reading experience and creates a cohesive warmth. AllTrails uses grey for these labels. Komoot uses grey. Grey is generic. Low-opacity brand green is specific.

---

**PM-5: Walk cards that separate image from content with a hairline brand-green rule.**

In the walk list and walk card components, add a `2px` or `3px` solid `#2C4A14` top border to the card body (the white section beneath the image). This thin brand-green rule acts as a visual anchor between the image band and the content, and is a strong structural signature. It appears on every card in every context. At a grid level, the repeated green rules create a visual rhythm that is immediately recognisable as Sniffout.

```css
.walk-card-body, .walk-list-card-body {
  border-top: 2px solid #2C4A14;
}
```

This is a direct response to AllTrails' approach of overlaying text on photos - Sniffout cleanly separates image from content, and marks that separation with its brand colour.

---

## 6. What to keep

**The background `#F7F5F0`.** This warm off-white places Sniffout in the same register as Butternut Box and Headspace - consumer-warm, not utility-grey. It is correct and distinctive within the outdoor app space where most apps use pure white or dark surfaces.

**Plus Jakarta Sans as the body and UI font.** It is genuinely excellent for legibility at small sizes and has enough variation in weight to handle the full hierarchy. It should not change, only be supplemented at display sizes with Fraunces.

**The brand green `#2C4A14`.** It does not need to change. It simply needs to be used with more commitment and consistency.

**The sticky header with the wordmark and "Get the app" pill.** It is clean, unobtrusive, and the wordmark-as-logotype approach is correct for this brand stage. The only suggested change is applying the sienna accent to the CTA pill to add personality.

**The seasonal notes component (amber + green pills).** Distinctive, informative, and directly tied to the dog-ownership use case. No other outdoor or walking site shows seasonal hazards in this form. It should be retained and potentially expanded in future rounds.

**The breadcrumb navigation.** Functional, clean, and correct for a content site. Keep as is.

**The card-with-band pattern introduced in the guides redesign spec.** Coloured header bands with category icons replace full-bleed photography and create a distinctive visual system for content cards. This is a structural differentiator from AllTrails' photo-only card treatment and should be considered for walk cards too as a fallback when photography is unavailable.

---

## Implementation priority

The five changes with the highest ratio of impact to effort, in order:

1. **Add Fraunces as H1 display font.** One Google Fonts import, one CSS font-family override on `h1` elements. Two hours of Developer time. Immediate and dramatic visual differentiation.

2. **Apply section labels in low-opacity brand green.** Change one CSS value on `.walk-section-heading` from `#888888` to `rgba(44,74,20,0.55)`. Ten minutes. Cohesion across all pages improves immediately.

3. **Add card top rule.** `border-top: 2px solid #2C4A14` on `.walk-list-card-body` and `.walk-card-h-body`. Ten minutes. Creates the signature card rhythm.

4. **Introduce sienna `#B85C2C` as accent token.** Add CSS token, apply to header CTA pill and walk difficulty "Moderate" badge. One hour. Adds the second personality colour to every page.

5. **Replace gradient hero walk backgrounds with a richer stop.** Update the linear-gradient on `.walk-hero`, `.walk-card-h-img`, `.walk-list-card-img` to anchor the darkest stop at `#2C4A14` rather than `#6E9048`. Creates stronger brand colour presence in image placeholders while photos are being sourced.

# Part 3: Specific UI Issues -- Owner Review

**Added:** 2026-03-29
**Scope:** Six specific issues flagged by the site owner across the homepage, walks index, and guides index. Each issue is assessed as either a quick CSS/HTML patch or a broader design pattern that requires a systemic fix.

---

## Issue UI-1 -- Sniffout logo duplicated inside the hero image (Homepage)

**What is happening:**
In `index.html` (lines 13-15), a 36px white SVG of the Sniffout paw-mark logo is rendered directly inside `.home-hero-inner`, above the H1. The logo appears on top of the hero photography. The sticky header already shows the full wordmark on every page.

**Assessment: Quick HTML fix. No design decision involved.**

The duplication weakens both instances. The hero logo competes with the heading copy and fragments the page hierarchy. The header wordmark is the correct and sufficient brand presence. The hero should lead with the headline.

**Fix:**
Remove the `<svg>` element and its closing tag from inside `.home-hero-inner` in `index.html`. The header wordmark carries the brand. The hero needs no second logo.

---

## Issue UI-2 -- "Know before you go" benefit icon is ambiguous (Homepage)

**What is happening:**
The "Know before you go" benefit block in `index.html` (lines 29-30) uses a custom SVG combining a sun circle, radiating rays, and a partial cloud arc (`M3 9a9 9 0 0 1 9-6 9 9 0 0 1 6 2.3`). The partial arc reads as an incomplete circle or loading indicator at small sizes. The icon does not clearly communicate either "weather" or "know ahead of time."

**Assessment: Quick SVG replacement. Minor design consideration: review all three benefit icons together.**

The icon needs to be immediately legible at 24px with no decoding required. The Lucide icon set has cleaner options.

**Primary recommendation: `umbrella`**

The umbrella silhouette is one of the most universally legible icons in any icon set. At 24px it is unmistakable. It communicates: weather awareness, preparedness before leaving, protection from conditions. This maps directly to "Know before you go." No competitor uses it as a primary UI element -- it is safe to own.

Lucide implementation:
```svg
<svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="#2C4A14" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
  <path d="M23 12a11.05 11.05 0 0 0-22 0zm-5 7a3 3 0 0 1-6 0v-7"/>
</svg>
```

**Secondary recommendation: `thermometer`** -- if the emphasis is on conditions data (temperature, heat risk) rather than weather preparedness. The thermometer reads as "checking conditions" and has relevance to the heat warning use case. Less universally linked to "going outside" but more specific to the weather-intelligence product angle.

**Design consideration:**

## Systemic fix for Issues UI-4, UI-5, and UI-6

Issues 4, 5, and 6 share a single root cause: **white-background structural containers that are full-bleed on mobile but become contained rectangles on desktop, without any border-radius applied at the desktop breakpoint.**

The design language uses `border-radius: 16px` on cards, `border-radius: 12px` on smaller blocks, and `border-radius: 20px` on chips. But the structural page containers (`.area-hero`, `.filter-bar`, `.area-sidebar`) predate this rounded language and were never updated to match it on desktop.

**There are two valid approaches:**

**Option A -- Apply border-radius at desktop breakpoint (recommended)**

Add `border-radius: 12px` to contained containers inside the desktop media query. This matches the card design language and makes the header sections feel like anchored components rather than raw browser defaults.

```css
@media (min-width: 768px) {
  .area-hero {
    border-radius: 12px;
  }
  .area-sidebar {
    border-radius: 12px;
    border-left: none;
    border: 0.5px solid rgba(0,0,0,0.08);
  }
}
```

For the guides filter bar: override the white background within the guides context to use the page background colour instead, eliminating the floating-rectangle appearance:

```css
.guides-page .filter-bar {
  background: transparent;
  border-bottom: none;
  padding-left: 0;
  padding-right: 0;
}
```

**Option B -- Remove background colours from structural containers and let them inherit the page background**

Simpler but loses the visual anchoring. The `#F7F5F0` page background would show through all containers. This works if the page background is consistent, but loses the "page header is a distinct card" quality that works well on the walks index.

**Recommendation: Use Option A for `.area-hero` and `.area-sidebar`, and Option B (transparent) for `.filter-bar` on the guides page.** The walks page header is a distinct section that benefits from a white card treatment. The guides filter bar is a utility row and should blend into the page.

This fix is categorised as a **Developer CSS patch** but should be tracked alongside the identity recommendations above, since the rounded-container treatment directly connects to the overall design language cohesion described in PM-5 (card top rules, consistent radius).
